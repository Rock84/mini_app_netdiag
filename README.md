# ⚡ NetDiag Micro (Agentless Network & RDP Checker)

Компактная утилита на базе FastAPI для безагентной диагностики сети между клиентом и Windows Server (VPS). 

Проект разработан для системных администраторов, которым необходимо быстро выявлять причины проблем с подключением (блокировки RDP, сбои DNS, перегрузка сервера) без установки дополнительного ПО на стороне клиента.

## 🚀 Особенности и выполняемые задачи

* **Определение работоспособности DNS:** Скрипт проверяет, способен ли клиент корректно разрешать доменные имена. При сбоях фиксируется ошибка связи.
* **Идентификация DNS-провайдера:** Автоматически определяет, какой именно DNS-сервер использует клиент (например, дефолтный сервер провайдера или публичные сервера Google/Cloudflare).
* **Диагностика доступности RDP (3389):** Использует механизм Timing Attack в браузере (через `fetch`), чтобы достоверно определить, блокирует ли локальный провайдер пользователя подключение к порту удаленного рабочего стола.
* **Алиби сервера (Server Metrics):** Автоматически фиксирует нагрузку на CPU и RAM сервера в момент проверки, исключая жалобы на "тормозящий VPS".
* **Reverse Ping & Tracert:** Сервер автоматически запускает обратную трассировку (без ограничения по прыжкам) к IP-адресу клиента для анализа трансатлантических маршрутов и потерь пакетов.

## 🛠 Установка и запуск

1. Клонируем репозиторий:
   ```cmd
   git clone [https://github.com/ВАШ_НИК/ИМЯ_РЕПОЗИТОРИЯ.git](https://github.com/ВАШ_НИК/ИМЯ_РЕПОЗИТОРИЯ.git)
   cd ИМЯ_РЕПОЗИТОРИЯ
   ```

2. Устанавливаем зависимости:
   ```cmd
   pip install -r requirements.txt
   ```

3. Открываем порт 8000 в брандмауэре Windows (запустить PowerShell от имени Администратора):
   ```powershell
   New-NetFirewallRule -DisplayName "NetDiag Port 8000" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
   ```

4. Запускаем сервер:
   ```cmd
   uvicorn netdiag_micro:app --host 0.0.0.0 --port 8000
   ```

## 📖 Как использовать

* **Для клиента:** Отправьте пользователю ссылку `http://IP_СЕРВЕРА:8000/client`
* **Для администратора:** Панель с готовыми отчетами доступна по ссылке `http://IP_СЕРВЕРА:8000/admin`
